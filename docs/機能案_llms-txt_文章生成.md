# llms.txt 文章生成 — インプットフィールド・プロンプト機械生成・AI 連携

llms.txt の役割が「思想・プロトコル」と決まっているため、**複数のインプットフィールドからプロンプトを機械生成**し、**OpenAI API Key をセットしていればそのプロンプトで AI 生成**する構成を想定する。

---

## llms.txt 周りの方針

1. **複数のインプットフィールド**で、プロンプトを機械生成する。
   - サイト種別、ストア名・ドメイン、補足キーワード、禁止事項プリセット、**参照する md ファイル（docs/ai 等）の指定** など。
   - 指定した md（例: 思想とプロトコル用プロンプトのテンプレ）もフィールドで選べるようにする。

2. **プロンプトは常に機械生成**し、画面に表示・コピー可能。ユーザーはそれを外部 AI に貼って自分で文章を取得してもよい。

3. **OpenAI API Key をアプリ設定でセットしていれば**、機械生成したプロンプトで API を呼び、**AI 生成した文章をレスポンスで受け取る**。プレビュー → 「採用」で **Files API により llms.txt の内容を更新**する。

4. **md ファイルの使用もフィールドで指定**する。例: 「プロンプトに含める doc」として `docs/ai/README.md` や自前の md の URL／内容を指定し、プロンプトに埋め込む。

---

## インプット例（フィールド案）

| フィールド | 内容 |
|------------|------|
| サイト種別 | コーポレート / EC のみ / コーポレート兼 EC（雛形と共通） |
| ストア名・ドメイン | 自動取得 + 任意で上書き |
| 補足キーワード | 自由文 1〜3 行 |
| 禁止事項 | チェックで複数選択（価格・在庫推測禁止 等） |
| 参照 md | プロンプトに含める md を指定（アップロード URL や「使用するテンプレ」選択） |

→ これらを **docs/llms-txt_思想とプロトコル_AI用プロンプト.md** の構造に合わせて組み立て、1 本のプロンプト文字列を生成。API Key ありならそのプロンプトで OpenAI を呼ぶ。

---

## llms.full.txt — 毎日自動生成

**llms.full.txt** は「サイト情報全部」なので、**ストアデータから毎日自動生成**する。

- **含めるもの**
  - **コレクションごとの商品**（コレクション名・商品一覧または要約）
  - **FAQ**（ストアの FAQ ページ or メタフィールド等から取得）
  - **ロケーション**（店舗・拠点情報）
  - **ブランド**（ブランド一覧・説明）
  - **配送・支払い**（ポリシー要約、対応決済・送料の要約）

- **実行**  
  日次バッチ（cron / Shopify Flow / スケジュールジョブ等）で、上記を API で取得し、1 本のテキストにまとめて **Files API で llms.full.txt を更新**（同一 URL で中身だけ差し替え）。

→ 思想は llms.txt、事実の一覧は llms.full.txt が毎日更新、という役割分担になる。

---

## まとめ

- **llms.txt**: 複数インプットフィールド → プロンプト機械生成。API Key セット済みならそのプロンプトで AI 生成し、採用で llms.txt 更新。md の使用もフィールドで指定。
- **llms.full.txt**: コレクションごとの商品・FAQ・ロケーション・ブランド・配送支払いを**毎日自動生成**。
