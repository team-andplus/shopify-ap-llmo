# llms.txt 文章生成 — インプットからプロンプト／API 連携

llms.txt の役割が「思想・プロトコル」と決まっているため、**アプリ内のインプット（サイト種別・ストア名・キーワード等）からプロンプトを組み立て、文章を用意する**機能が考えられる。

---

## 2 パターン

| パターン | やること | ユーザー体験 |
|----------|----------|----------------|
| **A. プロンプトを機械生成** | 入力フィールド（サイト種別・ストア名・補足キーワード等）から、**プロンプト文をテンプレートで組み立て**、画面に表示する。コピーボタンでクリップボードにコピー。 | ユーザーがそのプロンプトを ChatGPT / Claude 等に貼り、返ってきた文章を自分で llms.txt に反映する（案B の「コンテンツ→ファイル」で編集）。 |
| **B. API で文章を取得** | 上記と同様にプロンプトを組み立て、**OpenAI API（または他プロバイダ）をアプリから呼ぶ**。レスポンスの文章を受け取り、プレビュー表示 → ユーザーが「採用」すると **Files API で llms.txt の内容を更新**（既存ファイルがあれば中身を差し替え）。 | アプリ内で「生成」ボタン → 数秒で文章が表示 → 「この内容で llms.txt を更新」で完了。API キーはユーザー入力か、アプリ側で保持（有料オプション）か。 |

---

## インプットの例（役割が決まっているので組み立て可能）

- **サイト種別** … コーポレート / EC のみ / コーポレート兼 EC（雛形方針と共通）
- **ストア名・ドメイン** … `shop.name` / `shop.domain` は API で取得可能。任意で「ブランド名」を上書き入力。
- **補足キーワード** … 「扱い商品は〇〇」「誇張しない」「一次情報は商品ページ」など、1〜3 行の自由文。
- **禁止事項のプリセット** … 「価格・在庫の推測禁止」「在庫切れはサイトで確認と伝える」などをチェックで選択。

→ これらを **docs/llms-txt_思想とプロトコル_AI用プロンプト.md** のプロンプト構造に合わせて埋め込み、1 本のプロンプト文字列を生成する。

---

## 実装メモ

- **パターン A**  
  - フロントのみ。テンプレート（文字列 or 小さい JSON）でプロンプトを組み立て、表示・コピー。  
  - API キー不要。コストなし。思想は「ユーザーが外部 AI で生成」のまま。

- **パターン B**  
  - バックエンドに「プロンプト生成 → 外部 API 呼び出し → レスポンス返却」のルートを 1 本。  
  - 採用時は既存の「llms.txt 用 Files API 更新」の流れに乗せる（中身のテキストだけ差し替え）。  
  - **API キー**: ユーザーがアプリ設定で「OpenAI API キー」を入力して保存（暗号化して DB に保持）、またはアプリ提供側でキーを保持し利用料を課金。  
  - **プライバシー・利用規約**: 入力文・生成文をプロバイダに送る旨を明記。ストアデータ（商品名等）をプロンプトに含める場合は、ユーザー同意または匿名化の検討。

- **共通**  
  - 雛形の「サイト種別」と同一の選択肢にすると、雛形作成と「文章生成のたたき」が一貫する。

---

## まとめ

- **役割が決まっているので、インプットからプロンプトを機械的に生成できる。**
- **プロンプトだけ出す（A）** なら実装もコストも軽い。**API で文章を受け取り llms.txt を更新（B）** なら UX は良いが、API キー・利用料・プライバシーの設計が必要。
- まず **A** を実装し、必要なら **B** を「有料オプション」や「将来バージョン」で検討する、という段階でもよい。
