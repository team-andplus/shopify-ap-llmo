## 優先順位

### 1. llms.full.txt 強化 ⭐ 最優先

**現状の問題:**
- コレクション20件、商品30件/コレクションの制限
- 商品情報が title, vendor, productType, priceRange のみ

**対応:**

1. **全商品取得（ページネーション）**
   - GraphQL の `products` クエリで全件取得
   - first: 250 で繰り返し、cursor でページネーション
   - 手動生成ボタン: 上限500商品（待てる範囲）
   - 定時処理: 全商品取得（時間制限なし）

2. **商品情報追加**
   - `description`（商品説明）
   - `onlineStoreUrl`（商品URL）
   - `tags`（タグ）

3. **ストア情報追加**
   - 連絡先（メール、電話）
   - SNS リンク

4. **ポリシー追加**
   - プライバシーポリシー（`PRIVACY_POLICY`）
   - 利用規約（`TERMS_OF_SERVICE`）

**変更ファイル:**
- `app/lib/llmo-full.server.ts`

---

### 2. node-cron 定時処理

**実装:**
- `app/lib/cron.server.ts` を作成
- 毎日午前3時（JST）に実行
- 全ストアの llms.full.txt を生成
- ログローテート（5000件上限）

**処理フロー:**
```
1. DB から有効なストア一覧を取得
2. 各ストアについて:
   a. Admin API トークンで認証
   b. 全商品取得（ページネーション）
   c. llms.full.txt 生成
   d. Shopify Files にアップロード
3. ログローテート実行
```

**注意:**
- 1ストアずつ直列処理（並列だと重い）
- エラーハンドリング（1ストア失敗しても続行）
