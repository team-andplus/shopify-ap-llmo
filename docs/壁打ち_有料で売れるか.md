# 壁打ち：これ有料で販売して売れると思う？

開発方針（llms.txt = 思想・プロンプト機械生成・API Key あれば AI 生成、llms.full.txt = 毎日自動生成、など）はそのままで、「**有料で出して売れるか**」を壁打ちする。

---

## 率直に言うと

**今のタイミングで「有料だけ」でガンガン売れるかは、あまり楽観しにくい。** ただし「まったく売れない」とも言い切れない。**誰に・何を・いくらで**をはっきりさせて、無料枠と有料オプションを分けると、売れる可能性は出てくる。

---

## 売れにくい要因

1. **llms.txt の認知がまだ狭い**  
   ストアオーナーの多くは「llms.txt って何？」の段階。予算のライン項目として「AI 可視性」が通るケースはまだ少ない。

2. **競合の無料枠**  
   [LLMs.txtジェネレーター](https://apps.shopify.com/llms-txt)（Gokyo.ai）は無料プランあり。「とりあえず無料で試す」需要はそちらに流れやすい。

3. **効果が目に見えにくい**  
   SEO のように「順位が上がった」と数字で示しづらい。AI 検索での露出は、まだ測定・報告の習慣が少ない。

4. **思想・プロトコルは「わかる人に刺さる」**  
   差別化になる半面、思想を理解しない層には「ただのテキスト生成アプリ」に見え、安い競合で足りると思われがち。

---

## 売れる可能性がある要因

1. **「毎日 llms.full.txt 自動生成」は他に少ない**  
   コレクション・FAQ・ロケーション・ブランド・配送支払いを日次でまとめる機能は、他アプリではあまり見ない。**ここを有料の核**にできる。

2. **制作会社・代理店が「お客様向けの一式」として買う**  
   自社の思想で llms.txt を整え、複数ストアに導入する運用なら、月額を「1 ストアあたり数百円」にしても、件数でまとまれば売上になる。

3. **無料で入ってもらい、高度な機能で課金**  
   無料: プロンプト機械生成・llms.txt ひな形・head の link まで。有料: llms.full.txt 毎日自動生成、AI 生成（または API 利用量込み）、複数 md の管理など。**段階的に価値を見せてから課金**させる設計なら、有料化のハードルは下がる。

4. **導入支援 10〜20 万とセット**  
   README の「導入支援は 10〜20 万で提供」と組み合わせる。アプリ単体は低単価 or 無料、収益は「支援＋アプリ有料オプション」で取る。

---

## どうしたら売れる可能性が上がるか（選択肢）

| やり方 | 内容 | 売れる可能性 |
|--------|------|----------------|
| **A. 無料メイン＋有料オプション** | 基本は無料。llms.full.txt 毎日自動生成 or AI 生成（回数制限）を有料プランに。 | 無料で試してもらい、必要性が分かった層が課金。現実的。 |
| **B. 低単価で布石** | 月額 $2〜5 など安くして、「とりあえず AI 可視性やっておく」層を取りにいく。 | 単価は低いが、数が乗れば成立。競合の無料と「ちょっと上」の差別化が必要。 |
| **C. 制作会社向けにパック** | 「〇〇社推奨 llms.txt パック」のように、思想・雛形・支援をセットで販売。アプリ月額はパックに含む。 | 単体のアプリ販売というより、支援付きパッケージの一部。売上は支援単価で決まりやすい。 |
| **D. いったん無料で出してから有料化** | README どおり、まず無料で配布し、認知と利用が増えてから「llms.full.txt 自動生成は有料」など一部を有料化。 | 初期は収益よりシェア。将来の有料化で回収。 |

---

## 案：無料／有料の線引き（具体的案）

**プランは 2 本**: **無料プラン** と **有料 $7.5/月 プラン** のみ。

**無料プラン**
- llms.txt 生成用プロンプト作成（インプットフィールドで機械生成し、表示・コピー）
- head 差し込み（llms.txt / llms.full.txt / docs/ai への `<link>` 出力）
- （ひな形作成も無料に含める）

**有料プラン（$7.5/月）**
- llms.txt 生成に **AI 利用可能**（API Key をセットすると、プロンプトで API を呼び、生成文を採用で llms.txt に反映）
- **llms.full.txt の自動生成**（毎日。かつ **AI で補正**もする）

---

## 上記案の壁打ち

**良い点**
- **無料で「入り口」がはっきりしている。** プロンプト作成と head 差し込みだけで、llms.txt の存在と発見経路は確保できる。自分で ChatGPT 等に貼って文章を書く人には無料で完結。
- **有料の価値が 2 本柱で分かりやすい。** 「AI で llms.txt を書いてくれる」「llms.full.txt を自動で作ってしかも AI で整える」のどちらか／両方で課金理由になる。
- **llms.full.txt に「AI 補正」を入れる**のは差別化になる。生データの羅列ではなく、要約・トーンを整えたテキストにできる、と打ち出しやすい。

**補足・検討したい点**
- **無料版でも llms.txt の「ひな形」作成（Files API で 1 個つくる）は含めるか。** 含めると「プロンプト作る → 外部 AI で文章もらう → ひな形ファイルを編集して貼る」まで無料で完結。含めないと「ファイルは自分でアップロード」になる。**含めた方が無料の体験は良い**ので、ひな形作成は無料に含めてよいと思う。
- **有料の「AI で補正」**は、llms.full.txt の元データ（コレクション・FAQ 等）を集めたあと、**要約・文体の統一・冗長削除**などを 1 回 API に渡すイメージでよい。**トークンコストはユーザーの API Key でかかるので、アプリ側では特に気にしなくてよい。** 設計で決めるのは実行タイミング（毎日バッチの最後に 1 回など）だけでよい。
- **料金の出し方**: 有料は **1 本だけ（$7.5/月）**。llms.txt AI 利用と llms.full.txt 自動生成（AI 補正）の両方を含める。プランは無料と有料の 2 本立てでよい。

**文言の整理（そのまま使う場合）**
- 無料: 「llms.txt 用プロンプト作成」「head への link 差し込み」。オプションで「llms.txt 用ひな形ファイルの作成」も無料に含めると案内しやすい。
- 有料: 「llms.txt の AI 生成」「llms.full.txt の毎日自動生成（AI 補正付き）」。

---

## 有料オプション、いくらだと価値があるか（壁打ち）

**競合の目安**: [LLMs.txtジェネレーター](https://apps.shopify.com/llms-txt)（Gokyo）は Free の上に Silver **$4.99** / Gold **$9.99** / Diamond **$49.99**/月（USD）。コンテンツ量で段階化している。

**当アプリの有料の価値**
- llms.txt の **AI 生成**（ユーザー API Key 使用＝手間削減・ワンクリック）
- **llms.full.txt の毎日自動生成＋AI 補正**（他にあまりない）

→ 「トークンは自分持ち」なので、**アプリ料金は「便利さ・自動化の対価」**として見せる。

| 価格帯（月額 USD） | 価値の感じ方 | 想定 |
|--------------------|--------------|------|
| **$1.99〜2.99** | 安いが「有料の意味ある？」となりがち。 | とにかくハードルを下げたいとき。 |
| **$3.99〜5.99** | 「コーヒー 1 杯」程度。**手間が省ける＋full.txt が毎日更新**で「あり」と思いやすい。競合 Silver と同程度。 | 低めに出す場合。 |
| **$6.99〜7.99** | 手頃でありつつ「ちゃんとしたツール」感。**AI 補正付き full.txt が毎日**を前面に出せば「その価値はある」と言いやすい。 | **$7.5/月** はこの帯。SchemaBridge $15 の半額弱で、LLMO 有料の候補として妥当。 |
| **$8.99〜9.99** | しっかりしたツール感。 | 競合 Gold と並べる場合。差別化を説明できる層向け。 |
| **$10〜19** | ニッチな機能としては「高い」と感じるストアも。制作会社が**複数ストアでまとめて**使うなら許容されうる。 | 1 ストア単体より、支援パックや複数店舗向け。 |
| **$20+** | この機能だけでは説明が重くなる。**導入支援とセット**なら「パック価格の一部」として成立。 | 単体の月額としては避けた方が無難。 |

**結論（一言）**
- **無料プラン** と **有料 $7.5/月 プラン** の 2 本立て。
- 有料で「llms.txt AI 利用」と「llms.full.txt 毎日自動生成（AI 補正）」の両方をつける。$7.5 は SchemaBridge $15 の半額弱で、棲み分けしやすい。
- 日本円だと「月額 1,000 円前後」の案内になる。

## 別案：競合のような制限ベースの多段階プラン（壁打ち）

競合（Gokyo）のような価格体系を取る場合のイメージ。

| プラン | 価格 | 機能（目安） |
|--------|------|----------------|
| **無料** | $0 | 5 コレクション、50 商品（llms.full.txt に含める上限）、プロンプト作成・head 差し込み |
| **シルバー** | $4.99/月 | 250 商品、50 コレクション、250 ブログ、50 カスタムページ。7 日間無料体験。 |
| **ゴールド** | $9.99/月 | 2500 商品、500 コレクション、500 ブログ、500 カスタムページ。7 日間無料体験。 |
| **ダイヤモンド** | $49.99/月 | 無制限。7 日間無料体験。 |

**壁打ち**

**メリット**
- **競合と比較されやすい**。「Gokyo と同じような料金表」で、商品数・コレクション数で選べるので分かりやすい。
- **無料で「llms.full.txt の味」を見せる**ことができる。5 コレクション・50 商品までなら無料で自動生成し、それ以上は有料、という流れ。
- **7 日間無料体験**は Shopify で一般的。有料プランのハードルが下がる。
- **スケールに応じた課金**で、大規模ストアからは単価を拾いやすい。

**デメリット・検討点**
- **当アプリの差別化は「思想・プロトコル」と「AI 補正」**。制限ベースにすると「中身の量」で比べられ、競合と同じ土俵になる。
- **制限の実装が必要**。llms.full.txt 生成時に「プランごとの上限」をチェックし、超えたら upgrade を促す UI・メッセージが必要。
- **プランが 4 つ**になると、説明・サポート・Billing API の管理が重くなる。今は 2 本立てでシンプルにしている。
- **ブログ・カスタムページ**をどこまで llms.full.txt に含めるか、仕様を決める必要がある（現状の機能案はコレクション・商品・FAQ・ロケーション・ブランド・配送支払いが中心）。

**どうするか**
- **2 本立て（無料 / 有料 $7.5）のまま**で、差別化を「思想＋AI 補正」に寄せる方が、当アプリの立ち位置に合う。
- **競合に近づけたい**なら、無料で「5 コレクション・50 商品まで llms.full.txt に含める」を入れて「味見」させ、有料を $4.99 / $9.99 の 2 段階（または 1 段階 $7.5）にする、という折衷もあり。その場合は「AI 補正」を有料の全プランに入れるか、上位だけにするかも決める。

---

## まとめ（一言）

- **「有料だけ」で今すぐ大きく売れるかは、あまり楽観しにくい。** 認知・競合・効果の見えにくさが重い。
- **売れる可能性を上げるなら**: 無料で入り口を作り、**llms.full.txt 毎日自動生成**や**AI 生成**を有料オプションにする（A）。単体で売るなら低単価（B）。支援とセットでパックにする（C）。いったん無料で出してから有料化（D）。
- **開発方針はそのままでよい。** 実装したうえで、**無料／有料の線**を「どこで引くか」だけ後から決めても間に合う。まずは「使ってもらえるもの」を出し、反応を見てから有料プランと価格を固める、という段階でよいと思う。
- **線引きの具体案**: **無料プラン** = プロンプト作成 ＋ head 差し込み ＋ ひな形作成。**有料 $7.5/月 プラン** = llms.txt の AI 利用 ＋ llms.full.txt 自動生成（AI 補正付き）。プランはこの 2 本立てでいける。
