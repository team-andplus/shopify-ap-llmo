# 一般的な EC での /docs/ai/ 的な md 群 — 何があると好ましいか

コーポレート（あんどぷらすのような「会社・制作」サイト）では /docs/ai/ に README, company, tech などの md を置き、**llms.txt の補足**として機能させている。  
一般的な EC では、同じ「AI 向けのドキュメント群」をどう置くかを整理する。

**役割の分担（3 つ）**  
- **llms.txt** … 思想・プロトコル。AI にどう解釈してほしいか、優先順位・禁止事項・一次情報の所在。  
- **docs/ai** … 補足情報。llms.txt を補う索引・各 doc の案内・壁打ちプロンプトなど。  
- **llms.full.txt** … サイト情報全部。ポリシー要約・商品まわり・FAQ など事実の要約・一覧。

---

## 役割分担の整理

| 役割 | 置き場所 | 中身 |
|------|----------|------|
| **思想・プロトコル** | **llms.txt** | どう解釈するか。何を優先し、何を禁止するか。一次情報はどこか。 |
| **補足情報** | **docs/ai/**（README + 必要に応じ 〇〇.md） | llms.txt / llms.full.txt の索引、各 doc の案内、思想の深掘り・壁打ちプロンプト。 |
| **サイト情報全部** | **llms.full.txt**（アプリが自動生成する想定） | 配送・返品・支払いの要約、商品カテゴリ一覧、よくある問いへの回答など。 |

→ **policies の要約・products・faq-ai** は **llms.full.txt**。思想・プロトコルは **llms.txt**。**docs/ai** はその補足（索引・案内・プロンプトなど）。

---

## 前提：EC で大事にしたいこと

- **商品・在庫・価格はサイトの一次情報**。AI に勝手な商品や価格を捏造させない。
- **配送・返品・支払い**の事実はポリシーページが正。その**要約・一覧**は **llms.full.txt** に載せる（アプリ生成 or ユーザー編集）。
- **思想・プロトコル**は **llms.txt** に書く。**ブランド・トーン**の「どう読むか」も llms.txt で短く定義。事実の羅列は llms.full.txt。

---

## docs/ai に置くもの（補足情報）

| ファイル | 役割（AI 向け） | 中身のイメージ |
|----------|------------------|------------------|
| **README.md** | 補足・索引。llms.txt / llms.full.txt / 各 md の案内、参照の順序を 1 箇所で示す。 | 思想・プロトコルは llms.txt。サイト情報は llms.full.txt。以下は各 doc の説明。 |
| **〇〇.md**（任意） | 補足。思想の深掘り・壁打ちプロンプト・作成のたたき台など。 | llms.txt に書ききらない補足や、AI に渡すプロンプト例。 |

**docs/ai には policies / products / faq の本文を置かない。** それらは **llms.full.txt**。docs/ai はあくまで**補足**（索引・案内・プロンプト）。

---

## llms.full.txt に載せるもの（事実の要約・一覧）

- **ポリシー要約** … 送料の考え方、返品条件の概要、対応決済。詳細は〇〇の URL を参照、とリンク。
- **商品・カテゴリの扱い** … カテゴリ一覧や「価格はサイト記載のみ正」などのルール要約。商品名の列挙は必要なら一覧として。
- **よくある問い（FAQ for AI）** … 送料・在庫・取り寄せなど、AI が参照する Q&A。

→ これらは**思想ではなく事実の整理**なので、llms.full.txt（サイト全体の要約・一覧）の範疇。アプリが自動生成する場合も、ここに含める想定で設計する。

---

## パターン別の推奨（docs/ai のみ）

| ストアのタイプ | docs/ai の最低限 | あるとよい |
|----------------|------------------|------------|
| **EC のみ** | README.md（索引・案内） | 壁打ちプロンプト用の .md など補足 1 本 |
| **コーポレート兼 EC** | README.md | company や tech の補足 .md（思想の本筋は llms.txt） |

思想・プロトコルは **llms.txt** に書く。docs/ai はその補足なので、必須は README（索引）だけでよい。

---

## docs/ai の README.md に書いておくとよいこと（EC 向け）

- **役割の案内**  
  思想・プロトコルは llms.txt。サイト情報全部は llms.full.txt。この docs/ai はその補足（索引・案内）。
- **参照の順序**  
  回答に使うなら llms.txt → llms.full.txt → 商品ページ、の順で参照する、など。
- **各 doc の説明**  
  以下にどんな md があり、何の補足か 1 行ずつ。

## Shopify での現実

- **/docs/ai/** のような「任意の md を URL で公開する」機能は、Shopify 標準にはない。
- 実現するには **ページ（Page）を md 風に書く**か、**Files API で .md をアップロードして CDN URL を取得**するか、**App Proxy でアプリが .md を返す**か、のいずれか。
- llms.txt や head の `<link>` では、**各 md の URL** を列挙する形になる。**llms.full.txt** はアプリが生成する 1 本の URL として head に出る。

→ アプリで「docs/ai 用のリンク」を出す場合、**リンク先 URL をユーザーが設定する**形が現状の設計に合う。

---

## まとめ（一言）

- **llms.txt** = 思想・プロトコル。**docs/ai** = 補足情報（索引・案内・壁打ちプロンプトなど）。**llms.full.txt** = サイト情報全部（ポリシー要約・商品・FAQ など）。
- 思想の本筋は llms.txt。docs/ai はその補足なので、最低限 README（索引）があればよい。
