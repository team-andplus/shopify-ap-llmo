# llms.full.txt — 毎日自動生成の仕様

**llms.full.txt** = サイト情報全部。思想ではなく事実の要約・一覧なので、**ストアデータから毎日自動生成**する。

---

## 含める内容

| 項目 | 元データ | 出力イメージ |
|------|----------|----------------|
| **コレクションごとの商品** | Shopify Admin API: Collections + Products | コレクション名ごとに、商品タイトルまたは要約の一覧。 |
| **FAQ** | ストアの FAQ ページ、メタフィールド、またはアプリが持つ FAQ 設定 | Q&A 形式のテキスト。 |
| **ロケーション** | Locations API | 店舗名・住所・営業時間などの要約。 |
| **ブランド** | 商品の vendor / メタフィールド / ブランド一覧 | ブランド名・短い説明。 |
| **配送・支払い** | ポリシーページ、Shipping / Payment 設定 | 送料の考え方、返品概要、対応決済の要約。 |

→ これらを 1 本のプレーンテキスト（または Markdown）にまとめ、**llms.full.txt として Files API で作成・更新**する。URL は固定（既存ファイルがあれば中身だけ差し替え）。

---

## 実行タイミング

- **毎日 1 回**、決まった時刻にバッチ実行。
- 実装案: **cron**（サーバー）、**Shopify Flow**（対応範囲が許すなら）、**スケジュールジョブ**（アプリのバックエンドで setInterval / node-cron 等）、または **Shopify の Scheduled Job / Webhook** のいずれかでトリガー。

---

## 技術メモ

- **Files API**: 既存の llms.full.txt 用ファイル ID をメタフィールド等で保持し、`fileUpdate` で中身を更新。初回は `stagedUploadsCreate` → `fileCreate` で新規作成し、返ってきた URL をメタフィールドに保存。
- **データ取得**: Storefront API または Admin API（GraphQL/REST）でコレクション・商品・ロケーション・ポリシー等を取得。FAQ はストアがページやメタで持っている場合、Scrape または API で取得する必要あり（FAQ アプリ連携があればその API）。
- **サイズ**: 50KB 以下を推奨（llms.txt 仕様）。商品数が多い場合は「直近 N 件」「コレクションあたり M 件」で打ち切り、または要約のみにする。
- **AI 補正**: 有料オプションで「AI で補正」する場合はユーザーの API Key で呼ぶ想定。**トークンコストはユーザー負担のため、アプリ側では特に気にしなくてよい。**

---

## まとめ

- llms.full.txt は **コレクションごとの商品・FAQ・ロケーション・ブランド・配送支払い** を毎日自動生成。
- 思想は llms.txt、事実の一覧は llms.full.txt が日次更新、という分担。
